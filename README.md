O projeto foi criado com foco em inteligência artificial, automação e interação humano-computador, simulando o comportamento de assistentes inteligentes.

# Detroit-Virtual-Assistant
🧠 Detroit is a virtual assistant developed in Python capable of listening to voice commands, responding via audio, and controlling computer applications. The project was created with a focus on artificial intelligence, automation, and human-computer interaction, simulating the behavior of intelligent assistants.

---

# ⚙️ Como ela funciona / How it works

---

Detroit opera em um ciclo contínuo onde executa quatro etapas principais:

1️⃣ Captura de Voz

A assistente permanece constantemente ouvindo o microfone do usuário utilizando um sistema de reconhecimento de fala.

Quando o usuário fala, o áudio capturado é convertido em texto, permitindo que o programa interprete o comando.

2️⃣ Interpretação do Comando

Após transformar a fala em texto, Detroit analisa a frase dita pelo usuário para identificar palavras-chave e determinar qual ação deve ser executada.

Exemplos de comandos reconhecidos:

Perguntar que horas são

Abrir o Google

Abrir o YouTube

Abrir a calculadora

Outras interações com o sistema

3️⃣ Execução da Ação

Depois de reconhecer o comando, Detroit executa a ação correspondente utilizando funções do Python para interagir com o sistema operacional ou abrir páginas na internet.

Exemplos de ações realizadas:

Abrir sites no navegador padrão

Executar aplicativos do sistema

Buscar informações simples do sistema

4️⃣ Resposta por Voz

Toda ação realizada pela Detroit é acompanhada por uma resposta em voz, utilizando um mecanismo de text-to-speech.

Isso faz com que a assistente:

Confirme comandos executados

Informe as horas atuais

Avise quando um aplicativo ou site está sendo aberto

Exemplo de interação:

Usuário: Detroit, que horas são?

Detroit:
Agora são 21 horas e 34 minutos.
Usuário: Detroit, abra o YouTube

Detroit:
Abrindo o YouTube agora.
🔁 Interação Contínua

Detroit funciona em um loop contínuo de escuta, permitindo que o usuário execute vários comandos seguidos sem precisar reiniciar o programa.

Isso cria uma experiência mais natural, semelhante às assistentes virtuais modernas.

Detroit operates in a continuous loop where it performs four main steps:

Voice Capture

The assistant continuously listens to the user's microphone using a speech recognition system.
When the user speaks, the audio is captured and converted into text.

Command Interpretation

After converting speech to text, Detroit analyzes the command to identify keywords and determine the requested action.

Examples of recognized commands include:

Asking for the current time

Opening Google

Opening YouTube

Opening the calculator

General system interactions

Action Execution

Once a command is recognized, Detroit executes the corresponding action using Python system commands or web browser calls.

Example actions:

Launching websites in the default browser

Opening local applications

Fetching system information

Voice Response

Every action performed by Detroit is accompanied by a spoken response using a text-to-speech engine.

This means Detroit will:

Confirm commands

Speak the current time

Inform the user when an application or website is being opened

Example interaction:

User: Detroit, what time is it?

Detroit:
The current time is 21 hours and 34 minutes.
User: Detroit, open YouTube

Detroit:
Opening YouTube now.
🔁 Continuous Interaction

Detroit runs in a continuous listening loop, allowing the user to give multiple commands without restarting the program.
This creates a more natural assistant-like interaction similar to modern voice assistants.
